# TiDB Release Notes 写作简要指南

## 面向对象

## Release notes 写作原则

*   类别明确

*   表意清晰

*   用户视角

包括参考 GitHub issue

### 类别明确

Release notes 可分为以下 3 类：

| 类别                                           | 说明 | 典型示例 |
| :------------------------------------------- | :- | :--- |
| Compatibility change（兼容性更改）                  |    |      |
| Improvement（提升改进）或 Feature enhancement（功能增强） |    |      |
| Bug fix（Bug 修复）                              |    |      |
